# Test Plan – ToDo Application

## Scope
- **UI:** Task creation form, task list, filters, completion toggle, delete action.
- **API:** CRUD operations on `tasks` resource.

## Strategy
- **Techniques:** Equivalence partitioning, boundary value analysis, exploratory testing (charters).
- **Focus:** Validation of required fields, error handling, consistency between UI and API.
- **Tools:** Web browsers (Chrome, Firefox), Postman for API testing, screenshots/logs for evidence.

## Entry Criteria
- Acceptance criteria defined and approved.
- Test environment available (demo ToDo app + API endpoints).
- Test data prepared.

## Exit Criteria
- 100% acceptance criteria covered by test cases.
- All critical and major defects resolved or deferred with justification.
- Test execution reports completed.

## Metrics
- Requirement coverage: X/Y (Z%).
- Defects by severity: Critical=0, Major≤1, Minor≤N.
- Test case execution status: Passed/Failed/Blocked.

## Risks
- Incomplete requirements may lead to gaps in coverage.
- Environment instability could block execution.
