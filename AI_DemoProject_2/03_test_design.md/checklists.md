# Checklists – ToDo Application

## Form Validation
- Required fields are enforced (Title).
- Maximum length validation (Title ≤ 80 characters).
- Clear error messages displayed near the field.
- Special characters handled correctly.

## UI Stability
- Buttons disabled or debounced to prevent duplicate actions.
- Loading states visible during operations.
- No duplicate tasks created on double-click.

## API Consistency
- Response codes follow standards (200, 201, 400, 404, 204).
- JSON schema consistent across endpoints.
- Error messages are descriptive and user-friendly.

## Accessibility
- Tab order is logical and consistent.
- Visible focus indicators on interactive elements.
- Text contrast ratio ≥ 4.5:1.
- Labels provided for all input fields.
